# Contributing to Let's Learn Venn

Thanks for your interest in contributing! 🙌

## Core Principles

- Keep the math **correct and verifiable**
- Keep the UI **clear, beautiful, and responsive**
- Prefer **readable code** over clever tricks
- Always test new expressions in the visualizer before submitting

---

## Report Bugs

Open a GitHub Issue and include:

- What you typed / clicked
- What you expected to see
- What actually happened
- Screenshot if possible

---

## Suggest Features

Open a GitHub Issue and explain:

- What you want to add
- How it helps in understanding Set Theory

---

## High-Impact Contribution Areas

- **New set operations** — e.g. power set visualization
- **More color themes** for the workspace
- **Mobile touch improvements**
- **Accessibility** — keyboard navigation, screen reader labels
- **More examples** in the Guide section (e.g. De Morgan visual proof)
- **4-set Venn diagram** support (ellipse-based)

---

## Pull Request Flow

1. Fork the repository
2. Create a feature branch: `git checkout -b feat/my-feature`
3. Make your changes
4. Run `npm run build` to verify it compiles
5. Open a PR with a clear description and screenshots
